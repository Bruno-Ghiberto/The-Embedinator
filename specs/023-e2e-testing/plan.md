# Implementation Plan: Comprehensive E2E Testing

**Branch**: `023-e2e-testing` | **Date**: 2026-03-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/023-e2e-testing/spec.md`

## Summary

Validate The Embedinator end-to-end from a clean TUI installation through every page, feature, and integration. This is a **testing and verification** spec -- no new features are built. The orchestrator executes 11 sequential phases of hybrid testing (automated + manual), documenting every finding in a real-time execution log. Bugs found during testing trigger an impasse protocol (halt, triage, fix, verify, resume).

## Technical Context

**Language/Version**: Python 3.14+ (backend), TypeScript 5.7 (frontend), Go 1.24+ (TUI installer), Rust 1.93.1 (ingestion worker)
**Primary Dependencies**: FastAPI >=0.135, LangGraph >=1.0.10, Next.js 16, React 19, Qdrant >=1.17.0, Docker Compose v2
**Storage**: SQLite WAL (`data/embedinator.db`), Qdrant (vector search), localStorage (frontend sessions)
**Testing**: smoke_test.py (13 automated checks), Playwright (1 live E2E test), Manual browser testing (user-guided)
**Target Platform**: Linux (Fedora 43, x86_64) with NVIDIA RTX 4070 Ti via native Docker Engine
**Project Type**: Testing & verification workflow (hybrid automated + manual)
**Performance Goals**: Chat response < 30s with GPU, each phase < 30 minutes, total E2E < 5 hours
**Constraints**: Docker Desktop cannot pass GPU on Linux (must use native Engine), docker group not propagated (use `sg docker -c`), `credsStore` must be `""` (not `"desktop"`)
**Scale/Scope**: Single user, single machine, 11 phases, 54 functional requirements, 91 individual checks

## Execution Model

This spec uses a **single orchestrator + user** model, NOT parallel agent waves.

```
Orchestrator <-> User (browser)
     |
     +-- Phase 0: Pre-flight -------- AUTOMATED (orchestrator checks environment)
     +-- Phase 1: TUI Install ------- HYBRID (user runs TUI, orchestrator verifies)
     +-- Phase 2: Backend API ------- AUTOMATED (orchestrator hits all endpoints)
     +-- Phase 3: Navigation -------- HYBRID (user navigates, orchestrator checks console)
     +-- Phase 4: Collections ------- HYBRID (user CRUDs via UI, orchestrator verifies storage)
     +-- Phase 5: Ingestion --------- HYBRID (user uploads, orchestrator verifies chunks)
     +-- Phase 6: Chat & RAG -------- HYBRID (user chats, orchestrator verifies stream + traces)
     +-- Phase 7: Settings ---------- HYBRID (user configures, orchestrator verifies persistence)
     +-- Phase 8: Observability ----- HYBRID (user views dashboards, orchestrator verifies data)
     +-- Phase 9: Edge Cases -------- HYBRID (orchestrator sends bad requests, user triggers UI errors)
     +-- Phase 10: Acceptance ------- HYBRID (orchestrator runs smoke + Playwright, user final walkthrough)
```

**Critical rule**: Orchestrator MUST NOT advance phases without explicit user confirmation at each gate.

## Phase-by-Phase Plan

### Phase 0: Environment Pre-flight (AUTOMATED, ~5 min)

**FR**: FR-001 -> FR-004 | **SC**: SC-001

Verify: Docker Engine type, GPU accessibility, credsStore setting, port availability, stale data cleanup, Go binary availability.

All checks run as bash commands via the orchestrator. Auto-advance if all pass.

### Phase 1: TUI Installation (HYBRID, ~10 min)

**FR**: FR-005 -> FR-008 | **SC**: SC-001, SC-002

User runs `./embedinator` or `./embedinator setup`, completes wizard. Orchestrator verifies 4 containers healthy, health endpoints respond, models available.

### Phase 2: Backend Health & API (AUTOMATED, ~5 min)

**FR**: FR-009 -> FR-012 | **SC**: SC-002

Hit all API endpoints, verify response structure, check DB schema, verify Qdrant reachability.

### Phase 3: Frontend Navigation (HYBRID, ~15 min)

**FR**: FR-013 -> FR-017 | **SC**: SC-003, SC-004

Playwright navigates all pages, captures screenshots. User navigates in browser, confirms visual integrity, tests dark mode, sidebar, command palette.

### Phase 4: Collection Management (HYBRID, ~10 min)

**FR**: FR-018 -> FR-022 | **SC**: SC-004

User creates/deletes collections via UI. Orchestrator verifies in SQLite + Qdrant.

### Phase 5: Document Ingestion (HYBRID, ~15 min)

**FR**: FR-023 -> FR-029 | **SC**: SC-004, SC-005

User uploads PDF, MD, TXT files. Orchestrator polls ingestion status, verifies chunks in Qdrant, tests re-ingestion.

### Phase 6: Chat & RAG Pipeline (HYBRID, ~20 min) -- MOST CRITICAL

**FR**: FR-030 -> FR-038 | **SC**: SC-005

User chats, evaluates response quality and citations. Orchestrator verifies NDJSON stream events, trace recording, session persistence. Known issues documented.

### Phase 7: Settings & Configuration (HYBRID, ~10 min)

**FR**: FR-039 -> FR-042 | **SC**: SC-006

User changes model, adds/removes API key. Orchestrator verifies persistence via API.

### Phase 8: Observability & Metrics (HYBRID, ~10 min)

**FR**: FR-043 -> FR-046 | **SC**: SC-007

User views dashboards after Phase 6 chat queries. Orchestrator verifies traces exist, stats non-zero.

### Phase 9: Edge Cases (HYBRID, ~10 min)

**FR**: FR-047 -> FR-050 | **SC**: SC-011

Orchestrator sends malformed requests, empty messages, invalid IDs. User triggers UI error states.

### Phase 10: Final Acceptance (HYBRID, ~15 min)

**FR**: FR-051 -> FR-054 | **SC**: SC-008 -> SC-013

Run smoke_test.py (13/13 target), Playwright workflow test, user final walkthrough. Generate acceptance report.

## Impasse Protocol

When a check fails:
1. **HALT** -- stop current phase
2. **LOG** -- append to logs.md (timestamp, check ID, severity, evidence)
3. **TRIAGE** -- classify BLOCKER/HIGH/MEDIUM/LOW (use Sequential Thinking for complex bugs)
4. **FIX** -- orchestrator fixes code (BLOCKER/HIGH only), user confirms
5. **VERIFY** -- re-run failing check
6. **RESUME** -- continue from next check

User must confirm BLOCKER/HIGH fixes before resuming. MEDIUM/LOW can be deferred.

## Environment Gotchas

| Gotcha | Fix |
|--------|-----|
| `credsStore: "desktop"` | Set to `""` in `~/.docker/config.json` |
| Alpine resolves `localhost` as IPv6 | Use `127.0.0.1` in healthcheck URLs |
| `output: "standalone"` bakes rewrites at build time | `BACKEND_URL` must be build ARG |
| `.env` file may not exist | Create if missing (can be empty) |
| Docker group not propagated | Use `sg docker -c "..."` wrapper |
| Stale Qdrant collections | Delete in Phase 0 |
