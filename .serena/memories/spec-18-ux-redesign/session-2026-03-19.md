# Spec 18 — UX/UI Redesign Context (Pre-Implementation)

## Status: PLANNED — Not started

## Stack
Next.js 16, TypeScript 5.9, Tailwind CSS 4.2, shadcn/ui

## Design Philosophy — "Intelligent Minimalism"
- Clean, spacious layouts with purposeful negative space
- RAG intelligence visible in UI: confidence indicators, citation highlights, meta-reasoning status
- Original identity — not a generic dashboard clone
- Collapsible sidebar nav (not just top nav bar)
- Command palette (Cmd+K) for power users
- Dark mode with system preference + manual toggle

## Current Frontend State (Spec 17 baseline)
- 5 pages: /chat, /collections, /documents/[id], /settings, /observability
- 21 hand-built components — NO shadcn, NO dark mode toggle, NO sidebar
- Fixed top navbar with 5 links
- Basic Tailwind classes, gray/blue palette, no design tokens
- SWR for data fetching, recharts for charts (already installed)
- Partial Radix primitives: @radix-ui/react-tooltip, dialog, select (already in package.json)
- No skeleton loaders, no empty states, no breadcrumbs, no responsive mobile menu
- globals.css: minimal — only 2 breakpoints, foreground/background CSS vars, Inter font

## Key Competitive Gap (vs rag-web-ui)
rag-web-ui has: shadcn/ui, dark mode, sidebar layout, skeleton loaders, responsive hamburger
Embedinator wins on: every backend capability — just needs frontend polish

## Page Redesign Priorities
1. **Chat** (hero) — streaming tokens + inline citation chips + confidence badge (5-signal expandable) + meta-reasoning status indicator + empty state
2. **Collections** — card grid with stats + search/filter + quick actions
3. **Documents** — file list with chunk preview panel + drag-and-drop zone
4. **Observability** — health cards + stage timing waterfall + polished charts
5. **Settings** — grouped sections + provider hub with connection status indicators
