## Critical Architecture Decision (R7): LangChain Factory on ProviderRegistry

Agent nodes across all 3 graph layers use LangChain BaseChatModel methods (ainvoke, with_structured_output, bind_tools) — NOT the LLMProvider ABC (generate, generate_stream). These are incompatible interfaces.

**Solution**: `get_active_langchain_model(db) -> BaseChatModel` added to ProviderRegistry.
- Returns ChatOllama, ChatOpenAI, or ChatAnthropic based on active provider
- Coexists with get_active_llm(db) -> LLMProvider for httpx streaming
- Wired via config["configurable"]["llm"] in chat.py
- nodes.py is UNTOUCHED — zero blast radius on agent layer

**Wiring gap discovered**: chat.py only passed {"configurable": {"thread_id": ...}} — never injected llm, tools, or reranker. The entire agent layer ran with llm=None.
