## Implementation

### T037 ‚Äî `backend/config.py`

In the `Settings` class, add:
```python
log_level_overrides: str = Field(default="", env="LOG_LEVEL_OVERRIDES")
```
Format: comma-separated `module.path=LEVEL` pairs.
Example: `backend.retrieval.reranker=DEBUG,backend.storage.sqlite_db=WARNING`

### T038 ‚Äî `backend/main.py` `_configure_logging()`

After `structlog.configure()`, parse the override string:
```python
import logging as stdlib_logging

override_map: dict[str, int] = {}
for pair in settings.log_level_overrides.split(","):
    pair = pair.strip()
    if not pair:
        continue
    if "=" not in pair:
        log.warning("http_log_override_invalid_format", pair=pair)
        continue
    module, level_str = pair.split("=", 1)
    level_int = stdlib_logging.getLevelName(level_str.strip().upper())
    if not isinstance(level_int, int):
        log.warning("http_log_override_invalid_level", module=module, level=level_str)
        continue
    override_map[module.strip()] = level_int
```

Pass `override_map` to `_filter_by_component` via closure (define the processor inside
`_configure_logging()` or pass it as a parameter).

### T039 ‚Äî `_filter_by_component` processor

The `_configure_logging()` processor chain (verified current state):
```python
processors=[
    structlog.contextvars.merge_contextvars,  # ‚Üê insert AFTER this
    structlog.processors.add_log_level,        # ‚Üê insert BEFORE this
    ...
]
```

Implement the processor:
```python
def _filter_by_component(logger, method_name, event_dict):
    component = event_dict.get("component", "")
    if not component:
        return event_dict  # no component key ‚Üí pass through
    override_level = override_map.get(component)
    if override_level is None:
        return event_dict  # no override ‚Üí pass through
    event_level = stdlib_logging.getLevelName(method_name.upper())
    if isinstance(event_level, int) and event_level < override_level:
        raise structlog.DropEvent()
    return event_dict
```

Use `method_name` (the log method called, e.g., `"debug"`, `"info"`) ‚Äî do NOT use
`event_dict.get("level")` because `add_log_level` has not run yet at this position.
